# 🔄 Search Index Sync System Guide

## المشكلة التي تم حلها

كانت هناك مشكلة في عدم تزامن البيانات بين:
- **قاعدة البيانات MongoDB**: تحتوي على العروض الحقيقية
- **ملف JSON**: يستخدم للبحث السريع

هذا كان يسبب:
- ✅ العروض المحذوفة تظهر في البحث
- ✅ العروض الجديدة لا تظهر في البحث
- ✅ التحديثات لا تنعكس على نتائج البحث

## الحل الجديد

### 1. **مزامنة تلقائية (Auto-Sync)**
- تتم تلقائياً عند إنشاء/تحديث/حذف العروض
- لا تحتاج تدخل يدوي
- تعمل في الخلفية

### 2. **مزامنة يدوية (Manual Sync)**
- متوفرة في لوحة الإدارة
- يمكن تشغيلها عند الحاجة
- مفيدة بعد استيراد البيانات

### 3. **مراقبة الحالة (Status Monitoring)**
- عرض حالة المزامنة في الوقت الفعلي
- إشعارات عند عدم التزامن
- إحصائيات مفصلة

## كيفية الاستخدام

### في لوحة الإدارة

1. **عرض حالة المزامنة:**
   - اذهب إلى لوحة الإدارة
   - ستجد قسم "Search Index Sync Status" في الأعلى
   - يعرض عدد العروض في MongoDB مقابل ملف البحث

2. **المزامنة اليدوية:**
   - اضغط على زر "Sync Now" إذا كانت البيانات غير متزامنة
   - أو "Force Sync" للمزامنة الإجبارية

### عبر Terminal/Command Line

```bash
# تشغيل المزامنة
node scripts/sync-offers.js

# أو باستخدام npm
npm run sync-offers
```

### عبر API

```bash
# الحصول على حالة المزامنة
GET /api/admin/sync-offers

# تشغيل المزامنة يدوياً
POST /api/admin/sync-offers

# مسح الـ cache
POST /api/admin/clear-cache
```

## المتى تحتاج المزامنة اليدوية؟

### ✅ **تتم تلقائياً في هذه الحالات:**
- إنشاء عرض جديد
- تحديث عرض موجود
- حذف عرض
- تغيير حالة العرض

### ⚠️ **تحتاج مزامنة يدوية في هذه الحالات:**
- استيراد البيانات مباشرة إلى MongoDB
- تعديل قاعدة البيانات خارج النظام
- فشل المزامنة التلقائية
- بعد استعادة النسخ الاحتياطية

## مراقبة الأداء

### في لوحة الإدارة:
- **MongoDB Offers**: عدد العروض في قاعدة البيانات
- **Search Index**: عدد العروض في ملف البحث
- **Status**: حالة التزامن (✅ In Sync / ⚠️ Out of Sync)
- **Last Sync**: آخر وقت مزامنة

### إشعارات:
- تحذير أحمر عند عدم التزامن
- زر "Sync Now" باللون الأحمر عند الحاجة للمزامنة
- رسائل نجاح/فشل واضحة

## الميزات الجديدة

### 🚀 **تحسينات الأداء:**
- تقليل وقت الـ cache من 5 دقائق إلى 1 دقيقة
- مزامنة في الخلفية لا تؤثر على الاستجابة
- تحديث تلقائي للبيانات

### 🔒 **الأمان:**
- المزامنة تتطلب صلاحيات المدير
- تشفير الاتصالات
- تسجيل العمليات

### 📊 **المراقبة:**
- لوحة تحكم مرئية
- إحصائيات مفصلة
- تحديث تلقائي كل 30 ثانية

## استكشاف الأخطاء

### المشكلة: العروض المحذوفة تظهر في البحث
**الحل:** 
1. اذهب إلى لوحة الإدارة
2. تحقق من حالة المزامنة
3. اضغط "Sync Now" إذا لزم الأمر

### المشكلة: العروض الجديدة لا تظهر
**الحل:**
1. انتظر دقيقة واحدة (وقت الـ cache)
2. أو اضغط "Force Sync" للتحديث الفوري

### المشكلة: خطأ في المزامنة
**الحل:**
1. تحقق من اتصال قاعدة البيانات
2. تحقق من صلاحيات الكتابة في مجلد `data/`
3. راجع سجلات الأخطاء في Console

## الملفات المهمة

- `lib/syncOffers.ts` - وظائف المزامنة الأساسية
- `pages/api/admin/sync-offers.ts` - API المزامنة
- `components/SyncStatus.tsx` - واجهة مراقبة الحالة

- `data/offers.json` - ملف البحث المتزامن

## ملاحظات النشر (Vercel)

- في بيئة Vercel، نظام الملفات في وقت التشغيل يكون للقراءة فقط، والاستثناء الوحيد هو مجلد `/tmp`.
- تمت تهيئة نظام المزامنة ليكتب الملفات إلى المسار: `/tmp/unlockvault/{offers.json, articles.json}`.
- جميع قراءات الاحتياطي في واجهات البحث والعروض تفضّل ملفات `/tmp` أولاً، ثم تعود إلى ملفات المستودع داخل `data/` إن لم تتوفر.
- ملفات `data/*.json` داخل المستودع تُستخدم أساسًا للتطوير المحلي، وهي نسخ ثابتة لا تُحدَّث في وقت تشغيل Vercel.
- إذا ظهر خطأ `EROFS: read-only file system`، فهذا يعني محاولة الكتابة داخل مجلد غير قابل للكتابة. الحل: تأكد أن عمليات الكتابة تستهدف `/tmp/unlockvault`.
